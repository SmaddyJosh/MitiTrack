================================================================================
MITITRACK: PROJECT SUMMARY & ARCHITECTURE OVERVIEW
================================================================================

1. FRONTEND (React & Vite)
--------------------------------------------------------------------------------
- Interactive Map View: Powered by the Google Maps JavaScript API via `@react-google-maps/api`. Displays a satellite base layer with coordinate search, custom region preset selection, and pins for executing AI scans.
- NASA FIRMS Active Fires Integration: Uses a custom toggle layer that queries the NASA GIBS (Global Imagery Browse Services) WMS (Web Map Service) endpoint to retrieve Near Real-Time (NRT) VIIRS thermal anomaly hotspot tiles across a 7-day rolling window, mapped dynamically onto the map's Web Mercator (EPSG:3857) projection.
- Analytics Dashboard: Visualizes metrics including:
  - KPI cards for scan credits, carbon assets, USD valuation, area monitored, and average risk.
  - A custom-built SVG sparkline trend graph showing chronological forest cover data.
  - A history data table with options to deep-link back to the map or delete logs.
- Internationalization: Supported in English, Spanish, French, Swahili, and German using `react-i18next`.

2. BACKEND (Node.js & Express)
--------------------------------------------------------------------------------
- Server Platform: Node.js Express server running on port 5000.
- Database: MongoDB (Atlas cluster) integrated via Mongoose schemas.
- User Model: Manages user accounts (passwords encrypted via `bcryptjs`), tracking remaining scan credits (quota system) and total credits earned (incentive system).
- ScanResult Model: Persists coordinate data, date, forest cover/deforestation percentages, classification counts, carbon metric calculations, soil suitability logs, and base64-encoded annotated result images.
- Security: Protected routes validated via JSON Web Token (JWT) verification middleware (`auth.js`).

3. API LAYER
--------------------------------------------------------------------------------
- Authentication API (`/api/auth`): Handles registration and login, returning JWT tokens and user profiles.
- Scans API (`/api/scans`):
  - `POST /`: Saves new scan results, deducts a user quota credit, and updates credit metrics.
  - `GET /`: Retrieves overall user scan history (optimized to omit heavy base64 strings).
  - `GET /scan/:id`: Fetches complete details of a single scan including the annotated image.
  - `GET /stats`: Generates aggregated stats and historical data trends for the dashboard.
  - `DELETE /:id`: Deletes a specific scan record from the database.
- AI Forest API (`/predict_map` / `/predict` / `/compare`): Python endpoints serving inference.

4. AI MODELS (YOLOv8 & Computer Vision)
--------------------------------------------------------------------------------
- Model Architecture: YOLOv8 custom-trained weights (`best.pt`) loaded via PyTorch and the `ultralytics` package.
- Target Classes: Performs object detection/segmentation on standard satellite imagery for classes: `Tree`, `Farmland`, `Building`, `Water`, and `Tin_Shade`.
- Analytical Metrics:
  - Deforestation Risk Score: Computed as `(1 - TreeCount / TotalDetections) * 100`.
  - Carbon Asset Estimation: Based on a standard capture heuristic of 22kg of CO2 per tree per year. Tons of CO2 captured is calculated as `(TreeCount * 22) / 1000`, valued at $40 per carbon credit.
  - Change Detection: Compare endpoint detects tree cover loss percentage, carbon credit degradation, and raises alarm states if forest loss exceeds 20%.

5. DEVELOPMENT TOOLS & LIBRARIES
--------------------------------------------------------------------------------
- FastAPI: Python micro-framework used to construct the machine learning service endpoints.
- Pillow (PIL): Image processing library used to draw colored bounding boxes and labels on prediction files.
- Google Static Maps API: Fetches high-resolution satellite image snapshots based on lat/lng coordinates and zoom levels for machine learning inference.
- NASA GIBS WMS: Dynamic web map layer for overlaying active wildfires.
